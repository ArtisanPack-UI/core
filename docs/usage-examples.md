# ArtisanPack UI Core - Usage Examples

This document provides practical examples of how to use the ArtisanPack UI Core package in real-world scenarios.

## Basic Setup Example

### Complete Setup Workflow

```bash
# Step 1: Install core package
composer require artisanpack-ui/core

# Step 2: Install additional ArtisanPack UI packages
composer require artisanpack-ui/cms-framework
composer require artisanpack-ui/visual-editor
composer require artisanpack-ui/ui-components

# Step 3: Publish base configuration
php artisan vendor:publish --tag=artisanpack-config

# Step 4: Scaffold all package configurations
php artisan artisanpack:scaffold-config
```

### Result Configuration File

After running the scaffold command, your `config/artisanpack.php` might look like:

```php
<?php
/**
 * ArtisanPack UI Unified Configuration.
 *
 * This file is automatically generated by the artisanpack:scaffold-config command.
 * It contains the default settings from all installed ArtisanPack UI packages.
 * You can safely modify these values to customize your application.
 */
return [
    'cms-framework' => [
        'settings' => [
            'site_title' => 'My Website',
            'items_per_page' => 20,
            'enable_search' => true,
        ],
        'theme' => [
            'name' => 'default',
            'color_scheme' => 'light',
        ],
    ],
    
    'visual-editor' => [
        'autosave_interval' => 300,
        'toolbar_style' => 'full',
        'upload_path' => 'uploads/editor',
    ],
    
    'ui-components' => [
        'theme' => 'modern',
        'enable_animations' => true,
    ],
];
```

## Configuration Usage Examples

### Example 1: CMS Framework Integration

**Service Provider Usage:**

```php
<?php

namespace App\Providers;

use Illuminate\Support\ServiceProvider;

class AppServiceProvider extends ServiceProvider
{
    public function boot()
    {
        // Get CMS configuration
        $cmsConfig = config('artisanpack.cms-framework.settings');
        
        // Share site title with all views
        view()->share('siteTitle', $cmsConfig['site_title'] ?? 'Default Site');
        
        // Set pagination defaults
        if (isset($cmsConfig['items_per_page'])) {
            config(['app.pagination.per_page' => $cmsConfig['items_per_page']]);
        }
    }
}
```

**Controller Usage:**

```php
<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class BlogController extends Controller
{
    public function index(Request $request)
    {
        // Get pagination setting from unified config
        $perPage = config('artisanpack.cms-framework.settings.items_per_page', 15);
        
        // Use the setting
        $posts = Post::paginate($perPage);
        
        return view('blog.index', compact('posts'));
    }
    
    public function search(Request $request)
    {
        // Check if search is enabled
        $searchEnabled = config('artisanpack.cms-framework.settings.enable_search', false);
        
        if (!$searchEnabled) {
            abort(404, 'Search functionality is disabled');
        }
        
        // Perform search...
    }
}
```

### Example 2: Visual Editor Configuration

**Blade Template Usage:**

```php
@php
    $editorConfig = config('artisanpack.visual-editor');
@endphp

<div class="visual-editor" 
     data-autosave="{{ $editorConfig['autosave_interval'] }}"
     data-upload-path="{{ $editorConfig['upload_path'] }}"
     data-toolbar-style="{{ $editorConfig['toolbar_style'] }}">
    <textarea name="content" id="editor">{{ $content ?? '' }}</textarea>
</div>

<script>
document.addEventListener('DOMContentLoaded', function() {
    const config = @json($editorConfig);
    
    // Initialize editor with configuration
    const editor = new VisualEditor('#editor', {
        autosaveInterval: config.autosave_interval * 1000, // Convert to milliseconds
        uploadPath: config.upload_path,
        toolbarStyle: config.toolbar_style
    });
});
</script>
```

**JavaScript Configuration:**

```php
// In your controller
public function edit($id)
{
    $post = Post::findOrFail($id);
    $editorConfig = config('artisanpack.visual-editor');
    
    return view('posts.edit', compact('post', 'editorConfig'));
}
```

```javascript
// In your JavaScript file
class EditorManager {
    constructor(editorConfig) {
        this.config = editorConfig;
        this.setupEditor();
        this.setupAutosave();
    }
    
    setupEditor() {
        this.editor = new Editor({
            toolbar: this.config.toolbar_style,
            uploadUrl: `/uploads/${this.config.upload_path}`
        });
    }
    
    setupAutosave() {
        if (this.config.autosave_interval > 0) {
            setInterval(() => {
                this.autoSave();
            }, this.config.autosave_interval * 1000);
        }
    }
    
    autoSave() {
        // Auto-save logic here
        console.log('Auto-saving content...');
    }
}
```

### Example 3: UI Components Theme System

**Layout Integration:**

```php
{{-- resources/views/layouts/app.blade.php --}}
@php
    $uiConfig = config('artisanpack.ui-components');
    $theme = $uiConfig['theme'] ?? 'default';
    $animationsEnabled = $uiConfig['enable_animations'] ?? true;
@endphp

<!DOCTYPE html>
<html lang="{{ str_replace('_', '-', app()->getLocale()) }}">
<head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>{{ config('artisanpack.cms-framework.settings.site_title', 'My App') }}</title>
    
    {{-- Theme-specific CSS --}}
    <link href="{{ asset("css/themes/{$theme}.css") }}" rel="stylesheet">
    
    @if($animationsEnabled)
        <link href="{{ asset('css/animations.css') }}" rel="stylesheet">
    @endif
</head>
<body class="theme-{{ $theme }} {{ $animationsEnabled ? 'animations-enabled' : 'animations-disabled' }}">
    @yield('content')
</body>
</html>
```

**Component Usage:**


## Advanced Usage Examples

### Example 4: Custom Configuration Validation

```php
<?php

namespace App\Providers;

use Illuminate\Support\ServiceProvider;
use Illuminate\Support\Facades\Validator;

class ArtisanPackConfigProvider extends ServiceProvider
{
    public function boot()
    {
        $this->validateConfigurations();
    }
    
    private function validateConfigurations()
    {
        // Validate CMS Framework config
        $cmsConfig = config('artisanpack.cms-framework.settings', []);
        $validator = Validator::make($cmsConfig, [
            'site_title' => 'required|string|max:255',
            'items_per_page' => 'integer|min:1|max:100',
        ]);
        
        if ($validator->fails()) {
            throw new \InvalidArgumentException(
                'Invalid CMS Framework configuration: ' . 
                implode(', ', $validator->errors()->all())
            );
        }
        
        // Validate Visual Editor config
        $editorConfig = config('artisanpack.visual-editor', []);
        $validator = Validator::make($editorConfig, [
            'autosave_interval' => 'integer|min:30',
            'upload_path' => 'required|string',
        ]);
        
        if ($validator->fails()) {
            throw new \InvalidArgumentException(
                'Invalid Visual Editor configuration: ' . 
                implode(', ', $validator->errors()->all())
            );
        }
    }
}
```

### Example 5: Dynamic Configuration Loading

```php
<?php

namespace App\Services;

class ConfigurationService
{
    public function getPackageConfig(string $package, string $key = null, $default = null)
    {
        $configKey = "artisanpack.{$package}";
        
        if ($key) {
            $configKey .= ".{$key}";
        }
        
        return config($configKey, $default);
    }
    
    public function updatePackageConfig(string $package, array $values)
    {
        $currentConfig = config('artisanpack', []);
        $currentConfig[$package] = array_merge(
            $currentConfig[$package] ?? [],
            $values
        );
        
        // Write back to config file (in development)
        if (app()->environment('local')) {
            $this->writeConfigFile($currentConfig);
        }
    }
    
    private function writeConfigFile(array $config)
    {
        $configPath = config_path('artisanpack.php');
        $export = var_export($config, true);
        
        $content = "<?php\n\nreturn {$export};\n";
        file_put_contents($configPath, $content);
    }
}
```

### Example 6: Middleware Integration

```php
<?php

namespace App\Http\Middleware;

use Closure;
use Illuminate\Http\Request;

class ArtisanPackThemeMiddleware
{
    public function handle(Request $request, Closure $next)
    {
        // Get theme configuration
        $theme = config('artisanpack.ui-components.theme', 'default');
        
        // Share theme with views
        view()->share('currentTheme', $theme);
        
        // Add theme class to body
        view()->composer('*', function ($view) use ($theme) {
            $view->with('bodyClass', "theme-{$theme}");
        });
        
        return $next($request);
    }
}
```

## Testing Examples

### Example 7: Configuration Testing

```php
<?php

namespace Tests\Feature;

use Tests\TestCase;

class ArtisanPackConfigTest extends TestCase
{
    public function test_config_file_exists()
    {
        $this->assertTrue(file_exists(config_path('artisanpack.php')));
    }
    
    public function test_cms_framework_config_is_accessible()
    {
        $config = config('artisanpack.cms-framework');
        
        $this->assertIsArray($config);
        $this->assertArrayHasKey('settings', $config);
    }
    
    public function test_visual_editor_config_has_required_keys()
    {
        $config = config('artisanpack.visual-editor');
        
        $this->assertArrayHasKey('autosave_interval', $config);
        $this->assertArrayHasKey('upload_path', $config);
        $this->assertIsInt($config['autosave_interval']);
        $this->assertIsString($config['upload_path']);
    }
    
    public function test_scaffold_command_works()
    {
        $this->artisan('artisanpack:scaffold-config')
             ->expectsOutput('Scaffolding ArtisanPack UI configuration file...')
             ->assertExitCode(0);
    }
}
```
